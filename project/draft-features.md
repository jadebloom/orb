# Draft Features

## Nomad

1. Migrating the local database to a remote one or making an accessible dump from it.

## Books

1. Books tracking.

## Activity Domain CRUD

1. An activity domain can be defined with its own CRUD.
2. Each domain will have its own data and heatmap.
3. Each domain should be available in a menu.

## Activity Tracking

1. Each domain may have different activities.
2. Different states (ex: obsidian check inputs) are defined (w/ emojis).

## Goals

1. Goals can be defined with incremental progress.
2. Progress steps are defined as well.
